GAMEKIKI# Developers

    시작하기


    1. 앱을 등록합니다.

    • 새로운 앱을 만듭니다. 앱 만들기

    • 생성된 앱의 Application id 와 Application secret 을 메모합니다.

    • 미니게임 사용을 원하시는 경우에는 미니게임 등록 페이지를 참조하시기 바랍니다.

    2. KikiDiv 컴포넌트를 import 시킵니다.

    위젯을 사용시킬 컴포넌트에 KikiDiv 컴포넌트를 import 합니다.

    3. 현재 컴포넌트에서 생성되는 KikiDiv를 참조할 참조변수를 생성합니다.

    4. 렌더링될 kikiDiv와 attribute들을 아래와 같이 설정합니다.




    KikiDiv 컴포넌트 Attributes

    Parameter | Example | Required | Description
    — | — | — | —
    ref | {this.child} | Required | 콜백함수 호출을 위한 참조변수
    appId | “app_a7suAtYd00000” | Required | 어플리케이션 id
    appSecret | “7e6dd75adb0c2d16cb1a0451**“ | Required | app secret
    userId | “test” | Required | 사용자의 id
    mgId | “MGMAI_D5w38188” | Required | 미니게임 id
    kikiCallback | {this.kikiCallback} | Required | 로그인이후 실행되는 callback

    5. attribute 설정이후 kikiCallback 함수를 등록합니다.




    6. kikiCallback에서 KikiDiv.init 함수를 로그인페이지 파라미터와 함께 호출하시면 위젯이 나타납니다.

    • 로그인이 되어있다면 위젯 클릭 시 위젯이 정상적으로 작동됩니다.
    • 로그인 되어있지 않다면 위젯 클릭 시 로그인 페이지로 이동합니다.
    • 로그인 페이지 설정은 로그인 설정 을 참조해주세요.

    KikiDiv init 함수 파라미터

    Parameter Example Required Description
    loginCookie true Required 로그인 페이지 호출 flag 값
    true=세션값이 있으면 로그인 페이지호출 안함
    false= 세션값 상관없이 로그인 페이지 호출
    loginpage “./login” Required 로그인 페이지를 호출할 때 이동될 로그인 url

    7. 사용자 세션체크

    사이트 사용자의 로그인 세션과 키키 위젯 로그인 세션의 동기화를 위해서
    사용자의 세션이 유지되는지 체크합니다.

    사용자 세션이 종료되었다면 키키위젯의 세션또한 종료해야 합니다.

    키키위젯의 세션을 종료하려면 init의 loginCookie 파라미터를 false로 설정하면 됩니다.
