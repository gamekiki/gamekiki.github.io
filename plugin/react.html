    로그인연동

    로그인은 KikiDiv 컴포넌트를 사용하시거나 REST API를 직접 호출하셔서 연동하실 수 있습니다.

    KikiDiv 컴포넌트를 통한 로그인 연동

    1. KikiDiv 컴포넌트를 아래의 속성들과 함께 import 합니다.

    KikiDiv 컴포넌트 Attributes

    Parameter Example Required Description
    ref {this.child} Required 현재 컴포넌트에서 생성된 키키 컴포넌트
    appId “app_a7suAtYd00000” Required 어플리케이션 id
    appSecret “7e6dd75adb0c2d16cb1a0451**“ Required app secret
    userId “test” Required 사용자의 id
    mgId “MGMAI_D5w38188” Required 미니게임 id
    kikiCallback {this.kikiCallback} Required 키키 컴포넌트가 준비되었을 때 실행되는 콜백함수
    1. 키키 컴포넌트가 준비될 때 실행되는 kikiCallback을 설정하고 해당 함수에서 로그인 함수를 호출합니다.

    Login Response

    Name Type Description
    okFlag boolean 로그인 성공여부
    true=로그인 성공
    false= 로그인 실패
    res object 로그인 호출이후 키키서버로부터 받은 response 값
    1. 로그인 성공여부를 체크해서 성공 시 메인페이지로 이동합니다.

    REST API를 통한 로그인 연동

    KikiDiv 컴포넌트를 사용하지 않고 REST API를 직접 호출해서 로그인 연동을 하실 경우 아래를 따라하시면 됩니다.

    1. Login REST API 를 참조해서 로그인 토큰을 발급받습니다.
    1. 발급 받은 토큰을 split을 통해서 3등분 한 뒤에 토큰, appId, userId, mgId를 아래와 같은 방식으로 쿠키에 저장합니다.
    쿠키의 key값 Example Description
    kiki_wg_tk_fr eyJ0eXAiOiJKV1QiLCJ 토큰을 “.”을 기준으로 나누었을 때 첫번째 인덱스의 값
    kiki_wg_tk_sc eyJhcHBJZCI6ImFwcF9wYU5pREfeee0xZjAw 토큰을 “.”을 기준으로 나누었을 때 두번째 인덱스의 값
    kiki_wg_tk_tr uSRzQrCkQD-ZMxgLE 토큰을 “.”을 기준으로 나누었을 때 세번째 인덱스의 값
    kiki_wg_mgId “MGMAI_D5w38188” 미니게임 id
    kiki_wg_pId “app_a7suAtYd00000” 앱 id
    kiki_wg_uId “test” 로그인 사용자 id

    시작하기

    1. 앱을 등록합니다.

      • 새로운 앱을 만듭니다. 앱 만들기
      • 생성된 앱의 Application id 와 Application secret 을 메모합니다.
      • 미니게임 사용을 원하시는 경우에는 미니게임 등록 페이지를 참조하시기 바랍니다.

    2. KikiDiv 컴포넌트를 import 시킵니다.

      • 위젯을 사용시킬 컴포넌트에 KikiDiv 컴포넌트를 import 합니다.
    1. 현재 컴포넌트에서 생성되는 KikiDiv를 참조할 참조변수를 생성합니다.
    1. 렌더링될 kikiDiv와 attribute들을 아래와 같이 설정합니다.

    KikiDiv 컴포넌트 Attributes

    Parameter Example Required Description
    ref {this.child} Required 콜백함수 호출을 위한 참조변수
    appId “app_a7suAtYd00000” Required 어플리케이션 id
    appSecret “7e6dd75adb0c2d16cb1a0451**“ Required app secret
    userId “test” Required 사용자의 id
    mgId “MGMAI_D5w38188” Required 미니게임 id
    kikiCallback {this.kikiCallback} Required 컴포넌트 생성 이후 실행되는 callback
    1. attribute 설정이후 kikiCallback 함수를 등록합니다.
    1. kikiCallback에서 KikiDiv.init 함수를 로그인페이지 파라미터와 함께 호출하시면 위젯이 나타납니다.
      • 로그인이 되어있다면 위젯 클릭 시 위젯이 정상적으로 작동됩니다.
      • 로그인 되어있지 않다면 위젯 클릭 시 로그인 페이지로 이동합니다.
      • 로그인 페이지 설정은 로그인 설정 을 참조해주세요.

    KikiDiv init 함수 파라미터

    Parameter Example Required Description
    loginCookie true Required 로그인 페이지 호출 flag 값
    true=세션값이 있으면 로그인 페이지호출 안함
    false= 세션값 상관없이 로그인 페이지 호출
    loginpage “./login” Required 로그인 페이지를 호출할 때 이동될 로그인 url

    1. 사용자 세션체크

      • 사이트 사용자의 로그인 세션과 키키 위젯 로그인 세션의 동기화를 위해서 사용자의 세션이 유지되는지 체크합니다.
      • 사용자 세션이 종료되었다면 키키위젯의 세션또한 종료해야 합니다.
      • 키키위젯의 세션을 종료하려면 init의 loginCookie 파라미터를 false로 설정하면 됩니다.
